Мы продолжаем работу над OpenClaw-плагином для версии 2026.4.5.

Контекст:
- Мы специально НЕ используем api.registerTool, потому что runtime не подхватывал plugin tools стабильно.
- Вместо этого выбрали архитектуру:
  - plugin = упаковка, lifecycle, запуск локального API server, поставка skill
  - skill = объясняет модели, как обращаться к локальному API
  - вызов API должен идти через существующие стабильные инструменты, сейчас через exec + curl
- Цель: чтобы при запросах про Stakewise баланс агент обращался к локальному localhost API и возвращал данные пользователю в чат.

Что уже реализовано:
- Путь плагина:
  - .openclaw/workspace/plugins/stakewise-staking
- Название плагина:
  - stakewise-staking
- В плагине созданы файлы:
  - openclaw.plugin.json
  - package.json
  - index.ts
  - skills/stakewise-staking/SKILL.md
- package.json настроен под openclaw 2026.4.5
- В index.ts реализован локальный HTTP server на:
  - 127.0.0.1:5005
- При регистрации плагина сервер пытается стартовать
- Если порт уже занят, повторно сервер не стартует
- Реализованы роуты:
  - GET /health
  - GET /api/get-balance
- /api/get-balance сейчас возвращает mock JSON:
  - {"ok":true,"result":"[mock response]: Balance 0.1 ETH"}
- Добавлена команда:
  - /stakewise-reset
  которая перезапускает сервер
- Добавлен skill:
  - stakewise-staking
  который инструктирует модель при запросах вроде:
    - "Верни мне stakewise баланс для адреса 0x..."
  вызывать:
    - curl -sS "http://127.0.0.1:5005/api/get-balance?address=<ADDRESS>"
  затем парсить JSON и возвращать пользователю только поле result

Ожидаемое поведение:
- После установки и включения плагина, а также новой сессии, если пользователь пишет:
  - "Верни мне stakewise баланс для адреса 0x..."
  агент должен использовать skill, обратиться к локальному API и вернуть:
  - [mock response]: Balance 0.1 ETH
